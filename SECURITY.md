# Security policy

## Reporting security issues

Please report security vulnerabilities through GitHub issues or by contacting the repository owner directly. Do not open public issues for critical security problems if you are unsure about disclosure.

## Local-first privacy

DocFix Studio is designed to run locally. In the basic version, no documents are sent to paid APIs or external cloud services. Files are processed locally using Docling.

## Public demos

If you deploy a public demo of DocFix Studio, do not upload sensitive documents. Users should assume the host can inspect uploaded files unless the deployment explicitly states otherwise.

For private or sensitive documents, run DocFix Studio locally.
